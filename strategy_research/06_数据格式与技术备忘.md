# 数据格式与技术备忘

## 通达信 .day 文件格式

### 路径
- 上证: `C:\new_tdx_mock\Vipdoc\sh\lday\shXXXXXX.day`
- 深证: `C:\new_tdx_mock\Vipdoc\sz\lday\szXXXXXX.day`
- 代码前缀: 60xxxx=沪主板, 00xxxx=深主板, 30xxxx=创业板, 68xxxx=科创板

### 二进制格式 (每条32字节)
```
偏移  长度  类型    含义
0     4     I       日期 (YYYYMMDD 整数, 如 20260721)
4     4     I*100   开盘价 (除100得到真实价格)
8     4     I*100   最高价
12    4     I*100   最低价
16    4     I*100   收盘价
20    4     float   成交额 (金额, 非 volume!)
24    4     I       成交量 (股数)  ← 注意是24-28, 不是20-24
28    4     -       保留
```

**关键坑**: 成交量在偏移24-28, 不是20-24。20-24是成交额(float)。

### Python 读取示例
```python
import struct
def read_day(code):
    fp = f'C:\\new_tdx_mock\\Vipdoc\\sh\\lday\\sh{code}.day' if code[0] in '69' \
         else f'C:\\new_tdx_mock\\Vipdoc\\sz\\lday\\sz{code}.day'
    bars = []
    with open(fp, 'rb') as f: data = f.read()
    for i in range(0, len(data), 32):
        r = data[i:i+32]
        if len(r) < 32: break
        bars.append({
            'd': struct.unpack('I', r[:4])[0],
            'o': struct.unpack('I', r[4:8])[0] / 100.0,
            'h': struct.unpack('I', r[8:12])[0] / 100.0,
            'l': struct.unpack('I', r[12:16])[0] / 100.0,
            'c': struct.unpack('I', r[16:20])[0] / 100.0,
            'v': struct.unpack('I', r[24:28])[0]  # 成交量在24-28!
        })
    return bars
```

## 常见坑点

### 1. Heredoc 语法错误
Bash 中用 `cat << EOF` 写 Python 脚本容易出错。
**解决**: 用 Write 工具直接写文件, 不要用 heredoc。

### 2. 超时 (timeout 5m)
- 全市场遍历×多特征计算容易超时
- **解决**: 优化算法, 如只处理当日涨停股(不遍历全部成分股), 可从几分钟降到2秒

### 3. grep "conflicting matchers"
- 组合 `-n` 和某些正则会冲突
- **解决**: 用 Python 处理文本, 不用 grep

### 4. 集合推导式 TypeError: unhashable type: 'list'
- 复杂 set 推导式会失败
- **解决**: 先 `set(c for (c,dd) in sector_daily if dd==d)` 再迭代

### 5. stock_list() 返回空
- 未调用 `tdx_source.initialize()`
- **解决**: 必须先 initialize()
